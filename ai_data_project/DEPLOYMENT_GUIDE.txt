PERFORMANCE CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════════

DATASET SIZE RECOMMENDATIONS:

Small Datasets (< 1,000 rows):
✓ All models train instantly
✓ Typical time: 5-15 seconds
✓ Smooth user experience

Medium Datasets (1,000 - 10,000 rows):
✓ Most models fast
✓ Random Forest/SVM may take 30-60 seconds
✓ Good performance expected

Large Datasets (> 10,000 rows):
✓ Random Forest slower
✓ SVM may take 2-3 minutes
✓ Consider reducing random forest estimators if needed
  (Edit in ml_models.py: n_estimators=50 instead of 100)

MEMORY USAGE:

Typical Usage:
✓ Small: 50-100 MB
✓ Medium: 200-500 MB
✓ Large: 1-2 GB

All Sizes:
✓ Memory efficiently managed
✓ Sessions stored server-side
✓ No memory leaks expected

CONFIGURATION OPTIONS
═══════════════════════════════════════════════════════════════════════════════

To Adjust Model Complexity:

SVR (line ~59):

BEFORE: n_estimators=100

Train-Test Split:

BEFORE: test_size=0.2

PRODUCTION DEPLOYMENT
═══════════════════════════════════════════════════════════════════════════════

Using Gunicorn:

1. Install Gunicorn:
   pip install gunicorn

2. Run with Gunicorn:
   gunicorn ai_data_project.wsgi --bind 0.0.0.0:8000

3. Use supervisor or systemd for auto-restart

Using Docker:

1. Create Dockerfile:
   FROM python:3.10
   RUN pip install -r requirements.txt
   CMD ["python", "manage.py", "runserver", "0.0.0.0:8000"]

2. Build and run:
   docker build -t ml-app .
   docker run -p 8000:8000 ml-app

Using Nginx + Gunicorn:

1. Configure Nginx as reverse proxy
2. Run Gunicorn on port 8000
3. Nginx forwards requests to Gunicorn
4. Handles static files efficiently
